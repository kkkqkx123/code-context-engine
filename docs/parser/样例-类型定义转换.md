# 类型定义转换样例

## 概述

本文档简要展示如何将不同编程语言的类型定义转换为自然语言表示。

## 转换模板

```
Type definition {normalizedName} {docstringPart} {context}
```

## 样例1：TypeScript接口

### 输入代码

```typescript
/**
 * Represents a user in the system
 */
interface User {
    id: string;
    name: string;
    email: string;
    age?: number;
}
```

### 转换结果

```
Type definition User that does Represents a user in the system file user
```

---

## 样例2：TypeScript类型别名

### 输入代码

```typescript
/**
 * User ID type
 */
type UserId = string;

/**
 * User status enum
 */
type UserStatus = 'active' | 'inactive' | 'pending';
```

### 转换结果

```
Type definition UserId that does User ID type file user
Type definition UserStatus that does User status enum file user
```

## 样例3：Python类型注解

### 输入代码

```python
from typing import List, Dict, Optional

# User type alias
UserId = str

# User data type
UserData = Dict[str, Any]

# User list type
UserList = List[User]
```

### 转换结果

```
Type definition UserId file user
Type definition UserData file user
Type definition UserList file user
```

### 输入代码

```java

 * Generic container for items

}
```

```

```

## 转换要点

1. **命名规范化**：将类型名转换为自然语言
2. **文档字符串**：保留类型描述
3. **泛型参数**：保留泛型参数信息
4. **上下文**：包含文件路径

## 简化处理

类型定义通常比较简单，转换时：
- 保留类型名和描述
- 简化类型参数表示
- 不展开复杂的类型结构
